Overview

By late 2024, Toddle's web platform had successfully transitioned to Design System 2.0 — a mature, scalable, and modern experience. However, Toddle's mobile apps, used primarily by families and students, had not been updated for almost three years.

This gap created a fragmented cross-platform experience, outdated UI patterns, frequent usability complaints, and a decline in trust and app store perception. To address this, I led the Mobile App Revamp and built Toddle Mobile Design System 2.0.

Key Screens

Design System 2.0

The first priority was establishing a dedicated mobile design system to align mobile with Web 2.0 — ensuring every designer could work efficiently using shared patterns and components.

Home Page

01Replaced large widgets with compact modules to support future scalability, resulting in a more balanced and extensible homepage structure.

Profile Page

01Replaced confusing home icon back navigation with a standard back button for predictable interactions.
02Grouped profile photo, name, contact details, and edit option into a single profile card.
03Reduced visual bulk — more actions visible with a cleaner, easier-to-scan layout.

Announcements

01Clear visual patterns to differentiate new, unread, and read announcements.
02Added "Unread" filter so users can focus on new content without scrolling through older items.
03Updated header buttons from outlined to plain icons — cleaner and more scalable.

Calendar

01Month selection moved into the header with a dropdown, replacing multiple controls for more compact navigation.
02Added month breakers to clearly show timeline transitions while scrolling.
03Updated colour palette to softer, more mobile-friendly colours.

Attendance Dashboard

Redesigned the full attendance flow including overview, marking, excusals, and category-wise breakdowns for teachers, families, and students.

Search

01Moved search inside tabs wherever tabs are present — inline search keeps users in the same view, making it more natural to search within context.

Font Scaling & Accessibility

01Redesigned flows to support iOS text scaling up to XXXL and accessibility sizes A1–A5.
02Defined text scaling at component level — deciding which text scales and which is capped to prevent layout breakage.
